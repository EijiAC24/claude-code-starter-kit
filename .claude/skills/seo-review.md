# SEO/LLMO/AIO Analysis Skill

> コンテンツのSEO・LLMO（LLM Optimization）・AIO（AI Overview）スコアを分析

## Analysis Framework

izanami LLMO Analyzer準拠のスコアリング（合計140点満点）

---

## 1. 構造的要素 (40点)

### 見出し階層 (10点)
- [ ] H1が1つのみ存在する
- [ ] H2-H6が論理的な階層構造
- [ ] 見出しがコンテンツを適切に要約
- [ ] 見出しにキーワードが自然に含まれる

### 要約・メタ情報 (10点)
- [ ] 160文字以内のmeta description
- [ ] 導入部分で記事の要点を説明
- [ ] Open Graph / Twitter Card設定

### 機能・FAQ (10点)
- [ ] FAQセクションの有無（FAQ Schema対応推奨）
- [ ] Q&A形式でユーザーの疑問に回答
- [ ] 「よくある質問」等の明確なセクション

### タグ設定 (10点)
- [ ] 関連カテゴリタグの設定
- [ ] キーワードタグの設定
- [ ] タグがコンテンツを適切に分類

---

## 2. プロダクトアピール (30点)

### 価値提案 (10点)
- [ ] 「解決」「削減」「向上」等の価値キーワード
- [ ] ユーザーの課題・ペインポイントへの言及
- [ ] 具体的なベネフィットの提示

### ユースケース (10点)
- [ ] 具体的な使用シーンの記載
- [ ] ターゲットユーザーの明示
- [ ] 「〜な方におすすめ」等のセグメント

### ユーザーの声・証言 (10点)
- [ ] レビュー・体験談の引用
- [ ] 具体的な数値を含む成功事例
- [ ] 信頼性のあるソースからの引用

---

## 3. AI最適化要素 (50点)

### 構造化データ (10点)
- [ ] JSON-LD形式のSchema.org実装
- [ ] 適切なスキーマタイプ選択（Article, Product, FAQ等）
- [ ] breadcrumbリストの実装

### 視覚的コンテンツ (10点)
- [ ] 画像のalt属性が適切
- [ ] 図解・インフォグラフィック
- [ ] 動画コンテンツ（YouTube埋め込み等）
- [ ] コード例にシンタックスハイライト

### 料金・プラン情報 (10点)
- [ ] 具体的な価格の明記
- [ ] プラン比較表の提供
- [ ] 無料トライアル等の情報

### E-E-A-T評価 (10点)
| 要素 | チェックポイント |
|------|-----------------|
| **Experience** | 実際の使用体験、事例 |
| **Expertise** | 専門知識、技術的正確性 |
| **Authoritativeness** | 引用元、公式リンク、SNS |
| **Trustworthiness** | 透明性、更新日、連絡先 |

### 自然な表現 (10点)
- [ ] AI的な過度に構造化された文章を避ける
- [ ] 自然な会話調の文章
- [ ] 冗長な表現の排除
- [ ] 「〜することができます」→「〜できます」

---

## 4. 技術的SEO (10点)

### メタデータ (4点)
- [ ] title タグ（60文字以内）
- [ ] meta description（160文字以内）
- [ ] canonical URL設定
- [ ] hreflang設定（多言語の場合）

### コード品質 (3点)
- [ ] 技術記事：コード例の提供
- [ ] コードのシンタックスハイライト
- [ ] コピーボタンの実装

### リンク構造 (3点)
- [ ] 関連記事への内部リンク
- [ ] 権威あるソースへの外部リンク
- [ ] リンク切れがない

---

## 5. ユーザー体験 (10点)

### 読みやすさ (4点)
- [ ] 適切な段落分け（3-4文程度）
- [ ] 1行あたり40文字程度
- [ ] リスト・箇条書きの活用
- [ ] 重要部分の太字・ハイライト

### ナビゲーション (3点)
- [ ] 目次（Table of Contents）
- [ ] パンくずリスト
- [ ] 関連記事リンク

### アクセシビリティ (3点)
- [ ] 画像のalt属性
- [ ] 適切なコントラスト比
- [ ] キーボードナビゲーション対応

---

## スコア目安

| スコア | 評価 |
|--------|------|
| 🎯 **90点以上** | 素晴らしい！AI検索に最適化されています |
| ✨ **70-89点** | 良好です。少しの改善で完璧になります |
| 💪 **50-69点** | 基本はOK。改善の余地があります |
| 🔥 **50点未満** | 基礎から改善していきましょう |

## 改善提案テンプレート

分析後、以下の形式で改善提案を出力：

```markdown
## 改善提案

### 高優先度（+10点以上の改善が見込める）
1. [具体的な改善内容]
   - 現状: [問題点]
   - 推奨: [改善方法]
   - 期待効果: +○点

### 中優先度（+5-9点の改善）
1. ...

### 低優先度（+1-4点の改善）
1. ...
```

---

## 使用方法

1. 分析対象のURL/ファイルを指定
2. 各カテゴリでスコアリング
3. 検出要素をチェック
4. 改善提案を優先度順に提示
